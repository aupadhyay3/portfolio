Bear Maps


Description: This application has the following features:
1) Map rastering: provide images of appropriate resolution that cover a region given coordinates of a rectangular region of the world and the size of the web browser window.
2) Routing: given a start and destination latitude and longitude, provide street directions between those points.


Coding Skills: Java, A* Algorithm, KD Trees, Hash Maps, Priority Queue
Outcome: The map software renders the appropriate section of the map based on the input and provides street directions using methods that are predominantly log(N) runtime where N is the number of tiles/nodes in a given section of the map.
Reflection: This project allowed me to deeply understand KD Trees and the A* algorithm. The trickiest method in terms of the KD Tree class was the nearest method which returns the closest point to the inputted coordinates, taking O(log(n)) time on average, where N is the total number of points in the KD Tree. After going through a few examples of identifying the nearest point by hand, I was able to create a recursive algorithm for nearest which picked the optimal direction in the k-dimensional space where it would be most likely to find the nearest point. I improved the method to “prune” or not visit any branch that was guaranteed to return a point that wasn’t as close as the current closest point. Through building this map system, I was able to understand the practical application of a tree that partitions points in a k-dimensional space, making it easier to locate the nearest point or region on a map. The A* algorithm was key to building my artificial intelligence system that solves arbitrary state space traversal problems. Specifically, given a graph of possible states, the AI finds the optimal route from the start state to a goal state using A* and a well-defined heuristic. As I ran through more examples, it became clear that modifying the heuristic, or the estimated distance between two points, led to substantial improvements in A* generating the best path more efficiently. For example, if you know the given destination is due east, you don’t want your algorithm to traverse due west for any reason even if there is a nearby node. I used a minimum priority queue to store the vertex and the sum of vertex’s distance from the source plus the heuristic estimate from the vertex to the goal destination. Ultimately, this project was very helpful in incorporating multiple data structures in a practical way especially since it made me more appreciative of Google Maps which likely uses similar data structures and algorithms although obviously much more efficiently and effectively.
